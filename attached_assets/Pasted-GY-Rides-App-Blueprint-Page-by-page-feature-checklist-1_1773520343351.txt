GY Rides App Blueprint

Page-by-page feature checklist

⸻

1. Splash Screen

Purpose

First impression. Brand recognition.

What should appear
	•	GY Rides logo
	•	App name
	•	Short tagline:
	•	“Your local ride, anytime.”
	•	or “Transport made easy in Giyani.”
	•	Loading animation

Must-have features
	•	Fast load
	•	Clean branding
	•	Auto-redirect after 2–3 seconds

⸻

2. Welcome / Onboarding Screen

Purpose

Explain what the app does in a few seconds.

What should appear
	•	Illustration or image of ride booking
	•	Short slides:
	•	Book private rides
	•	Find taxi routes
	•	Share rides
	•	Send parcels
	•	Book clinic transport
	•	Buttons:
	•	Get Started
	•	Login

Must-have features
	•	Swipe onboarding cards
	•	Skip button
	•	Language option if possible:
	•	English
	•	Xitsonga

⸻

3. Sign Up Screen

Purpose

Allow new users to create accounts.

What should appear
	•	Full name
	•	Phone number
	•	Email
	•	Password
	•	Confirm password
	•	User type:
	•	Passenger
	•	Driver
	•	Sign up button
	•	“Already have an account? Login”

Must-have features
	•	OTP verification for phone number
	•	Error messages for invalid input
	•	Password visibility toggle

Unique extras
	•	Sign up with Google
	•	WhatsApp number verification

⸻

4. Login Screen

Purpose

Returning users sign in quickly.

What should appear
	•	Phone/email field
	•	Password field
	•	Forgot password
	•	Login button
	•	Sign up link

Must-have features
	•	Secure login
	•	Remember me
	•	OTP login option

⸻

5. Role Selection Screen

Purpose

Choose app mode.

What should appear
	•	Passenger
	•	Driver
	•	Fleet / Admin later if needed

Must-have features
	•	Simple large buttons/cards
	•	Save last selected mode

6. Passenger Home Screen

This is the main booking screen. It must feel powerful immediately.

What should appear
	•	Google Map full or half screen

	•	Search bar:
	•	“Where are you going?”
	•	Pickup field
	•	Destination field
	•	Ride type options:
	•	Private ride

	•	Taxi route
	•	Medical transport
	•	Parcel delivery
	•	Quick destination buttons:
	•	Home
	•	Work
	•	Hospital
	•	Masingita Mall
	•	Taxi Rank
	•	Bottom navigation bar

Must-have features
	•	Auto-detect location
	•	Manual location selection
	•	Map drag pin
	•	Saved places
	•	Fare estimate preview

Unique extras
	•	“Book on WhatsApp” button
	•	“Low network mode”
	•	Local landmarks instead of only formal addresses

7. Location Picker Screen

Purpose

Set pickup and dropoff accurately.

What should appear

Must-have features

Unique extras

	•	Voice-based place search later

8. Ride Options Screen

Purpose

After entering route, user chooses service type.

What should appear

For each option:

	•	Name
	•	Estimated fare
	•	ETA
	•	Passenger capacity

Ride options to show
	•	Private Ride
	•	Shared Ride
	•	Taxi Route
	•	Medical Ride
	•	Parcel Delivery

Must-have features
	•	Compare price by ride type
	•	Estimated arrival time
	•	Surge pricing if applicable
	•	Promo code field

Unique extras
	•	“Cheapest option”
	•	“Fastest option”
	•	“Best for elderly”
	•	“Best for luggage”

Core unique features you should prioritize first

If you want the app to feel special quickly, build these first:

Passenger side
	•	Local destination presets
	•	Taxi route page
	•	Shared ride option
	•	Medical transport page
	•	WhatsApp fallback booking
	•	Cash payment
	•	SOS and trip sharing

Driver side
	•	Online/offline toggle
	•	Instant ride requests
	•	Live navigation
	•	Earnings dashboard
	•	Verification badge

⸻

Best MVP version to build first

Don’t build all 32 pages at once. Start with:
	1.	Splash
	2.	Onboarding
	3.	Login/Signup
	4.	Passenger Home
	5.	Location Picker
	6.	Ride Options
	7.	Booking Confirmation
	8.	Searching Driver
	9.	Driver Found
	10.	Live Trip
	11.	Payment
	12.	Trip Completion
	13.	Ride History
	14.	Driver Registration
	15.	Driver Home

Then add:
	•	Taxi routes
	•	Shared rides
	•	Medical transport
	•	Parcel delivery
	•	WhatsApp fallback

Best “unique” identity for GY Rides

Your app should be known as:

“The local transport app for rides, taxi routes, shared trips, parcels, and medical transport.”
